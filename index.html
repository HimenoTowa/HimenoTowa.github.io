Python numpy 等三个包安装方法

Posted on 2020-07-20

1. numpy

先找到Python的安装位置

windows cmd中输入

where python

-> D:\Python\Python.exe

到其附近的Scripts文件夹中

去https://pypi.org/project/numpy/#files下载对应文件到Scripts文件夹

看清楚自己是32位还是64位，将这个文件下载到Scripts文件夹下面

在cmd命令行中输入：

pip install D:\Python\Scripts\numpy-1.17.2-cp37-cp37m-win_amd64.whl

2. matplotlib

pip install matplotlib

3. scipy

pip install scipy

1 open Atom
2 ctrl + ,
3 download following package
linter

linter-ui-default
gcc-make-run
(dowload TDM-GCC if you use Atom on Windows.
Then enter gcc-make-run‘s setting, edit make utility as mingw32-make )
4 shutdown 360/电脑管家






FAQ:

a键新建文件。键入test.cpp。
实例代码：

press f6 to compile and run.

PS： Atom运行c++代码，终端输出utf-8的中文会乱码，解决方法：atom里的setting->core->encode改成GBK就不会乱码了。
或者在这个文件的右下角点击UTF-8，改成GBK也可。

PS2：实时检测代码的语法错误：
三个包： linter ; linter-gcc ; linter-ui-default;
打开cmd用 apm install xxx 安装。
当按ctrl+s保存文件时检查语法错误。
关闭360/电脑管家。（解决卡顿，编译巨慢，甚至ra3启动延迟两分钟等问题，没错，就是红警3）

  • 随时检查：可以打开linter-gcc的设置中的linter on-the-fly.（但要停用Atom自带的autosave这个包。autosave的功能是重启Atom时项目文件还在，停用的话每次要重开项目文件）
  • 阻止底部弹窗：可以关闭linter-ui-default的设置中的Show panel.
  • 有和.cpp一致的最新的.exe文件时，不重复编译，直接运行：可以关闭gcc-make-run中的unconditional build。

出现问题：
linter on-the-fly不实时检测代码错误，甚至不显示代码错误。
解决：
停用autosave这个包
退出所有已经打开的文件，退出项目文件，文件夹名不要带中文，关闭atom，重启atom。

PS3：按了F6之后，等个七八秒才编译成功。linter on-the-fly也极度地卡，没法用。
解决：关闭360/电脑管家。
PS of PS3: 这个问题可真是困扰我好久咧。

PS4：若需要使用auto（c++11）
比如循环

在gcc-make-run里link libraries栏中加入 -std=c++11
