Atom c++ 环境

Posted on 2020-03-03

devc++用的好好的，不知为何突然就扯拐了，吃着火锅唱着歌就被劫了，源文件未编译，无法解决。
devc++确实很好用，界面简洁，反应迅速，十分Great，但有问题就很无奈。
只有换颜值更高的Atom了。
打开Atom，file->setting，下载gcc-make-run这个包即可。
如果下载不了，报npm cb() can’t callback，就打开cmd，自己输入apm install gcc-make-run，等会儿可看到atom里安装好了。
为什么是apm不是npm？因为我不知道什么时候用了淘宝镜像，就变成只能手动下载这个b样了。
重启atom，进入gcc-make-run的setting。
看这个包的说明可知：若为windows系统，需要下载TDM-GCC这个十分好用的C++ compiler，进入官网，下载64位版，一键安装next到底，它会很贴心地帮你把环境变量添加好。
继续按照说明，把这个包的setting里的make utility的内容改为：mingw32-make
到此，万事大吉。在文件里右键可以看到编译并运行的快捷键是F6。
#include <cstdio> int main() { printf("hello,world!"); return 0; }

以上实测可用。

//事实上踩了很多坑，但记录坑很没意思，心情都弄的很差，所以仅记录一本道的方法。

//淘宝镜像：

  1. 在 C:\Users\用户名.atom 目录下，新建文件名为 .apmrc 的文本文件，添加:
    registry=https://registry.npm.taobao.org/
    strict-ssl=false

  2. 通过命令行：npm config set registry https://registry.npm.taobao.org
    因为 apm 走 npm 源，所以需要更改 npm 源

  3. apm install –check // 检查安装环境

PS： Atom运行c++代码，终端输出utf-8的中文会乱码，解决方法：atom里的setting->core->encode改成GBK就不会乱码了。
